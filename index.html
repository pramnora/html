Create FREE Website using: Windows Azure/GitHub

  1. Set up Windows Azure to point to GitHub Folder/Files Repository

    • Sign in to Windows Azure a/c. online
    • authorize Azure to point to your GitHub a/c.
    • select a specific GitHub folder directory: html

    An Azure base URL address is created, from which it is possible to serve files stored inside of the GitHub folder you selected:

    • http://pmr1.azurewebsites.net

    NOTE: The above URL is pointing to the Github 'html' directory/only, 'invisibly'.

  2. Create files on the local machine

    Now, we are going to create 3 files: 2 x [.html]/1 x [.css] files inside of GitHub on our local machine.

    Open 'GitHub for Desktop' application on local machine; click 'show the html' repository inside of Explorer; then, use Windows Notepad editor to create 2 [.html] files:-

    • html
      • index.html
      • nextpage.html
      • css01.css

  3. Upload local machine files into GitHub online

    Next, we need to reflect the changes made on our GitHub local machine/to our GitHub online a/c.

    Open web browser; go to my GitHub online a/c.; find the 'html' folder repository; next, click on [Upload] files...

    ...select the 3 files to upload from off the local hard drive...using 'drag and drop' from Explorer window directly into GitHub online window.

  4. View uploaded website pages online

    Now, it's possible to view the 3 files you've previously uploaded by appending their file names to the Azure URL ending...

    • html
      • http://pmr1.azurewebsites.net/index.html
      • http://pmr1.azurewebsites.net/nextpage.html
      • http://pmr1.azurewebsites.net/css01.css

NOTE(1): [index.html] files are served up automatically; so, you don't need to type in the full file name; just state the base URL is enough.

NOTE(2): It's not necessary to go through step 2...create files on the local machine, first;
you could simply choose to create the files directly inside of GitHub online; and, then, proceed to step 4.

Another possible alternative, is you can simply surf to the base Azure URL address...

  • http://pmr1.azurewebsites.net

...and, from there just manually type in whichever filenames to go at the end of the URL.

NOTE: If there is no [index.html] available; then, viewers will be shown an error message: 'You do not have permission to view files'. In other words, only known filenames will be served.
